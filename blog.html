Common Questions

Here are some common questions that every web developer should know.

Q-1: What are the differences between map, filter, forEach and find?

Answer: There are some differences between map, filter, forEach and find. Below is about them:

map: The map method is applied to each element in an array and it returns a new array of the same length with the results.

filter: The filter method creates a new array with all elements that pass the condition by the provided function. It filters certain elements.

forEach: The forEach method executes a provided function once for each element in an array. It performs on each elements but does not create a new array.

find: The find method returns the value of the first element in an array that fulfills the provided function.

Q-2: What are the differences between Local Storage, Session Storage and Cookies?

Answer: There are some differences between Local Storage, Session Storage and Cookies. Below is about them:

Local Storage: Local Storage is a way of storing data on the client-side that persists even after the browser is closed. The data is stored in key-value pairs and can be accessed and modified using JavaScript.

Session Storage: Session Storage is a way of storing data on the client-side that is only available during the current browser session. It is cleared when the user closes the browser or the website that created it is closed.

Cookies: Cookies are stored on the client-side and are used to store data that persists even after the browser is closed. Cookies can store user login information. They has an expiration date, and is sent to the server with each HTTP request.

Q-3: What is Event-Loop? How does it work?

Answer:

The Event Loop is a process in JavaScript that allows for non-blocking execution of code by continuously checking a queue for pending tasks to be executed, moving them to the call stack for execution when the stack is empty.


When an event occurs in JS, such as a user clicking a button, the event is added to a task queue. The event loop continuously checks this queue for any pending tasks. If the call stack is empty, the event loop moves the next task in the queue to the call stack for execution. This process continues as long as there are tasks in the queue, allowing JavaScript to handle multiple tasks simultaneously without freezing or crashing.
